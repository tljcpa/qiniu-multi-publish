## 0. 文件用法说明

### 三类条目

| 类型 | 前缀 | 何时写 | 字段 |
|---|---|---|---|
| 决策 | **D-XX** | 拍板任何关键技术选型/超参数/架构取舍前 | 选项 / 选哪个 / 为什么这么做 / 为什么不那么做 / 改变这个决策的条件 |
| 踩坑 | **L-XX** | 遇到任何意外、bug、配置陷阱后 | 现象 / 原因 / 解决 / 教训 |
| 里程碑 | **M-XX** | 完成阶段性目标（如端到端跑通）| 完成内容 / 用时 / 下一步 |

### 写作要求
- **写"为什么"，不写"做了什么"**。"做了什么" git log 看得到，"为什么"只有这里有
- 一条 ≤ 6 行
- 不写废话，不补充客套
